Active Directory Lab – Implementation Steps

1. Domain Controller Deployment

I began by creating a virtual machine running Windows Server 2025 Datacenter Azure Edition in Microsoft Azure to serve as the Domain Controller.

After deployment, I installed Active Directory Domain Services (AD DS), promoted the server to a Domain Controller, and created a new forest: company.local.

2. Organizational Unit (OU) Structure

Under company.local, I created Branch 1 with Users, Computers, Servers, and Groups OUs. Inside Users, I created nested department OUs.

This structure improves organization, simplifies GPO application, and enables delegation of administrative control.

While creating the users, I also selected 'User must change password at next logon.' This allows users to safely set their own password, which only they should know.

3. Security Groups & RBAC

I created Security Groups to implement role-based access control (RBAC). Security Groups control access to resources such as file shares and GPO filtering.

Distribution Groups are used for email only and cannot assign permissions.

4. Adding Users to Security Groups

I added the user Amalia Lee to the security group B1_IT_Users to apply permissions at the group level.

5. Group Policy Object (GPO)

A Group Policy Object (GPO) is a collection of settings in Active Directory that can be applied to domains or Organizational Units (OUs) to centrally manage users and computers.

Computer Configuration

Applies to the computer itself and remains the same regardless of which user logs in.

User Configuration

Applies to user accounts and follows the user when they log in to a domain-joined device.

Policies vs Preferences

Policies cannot be changed by users, while Preferences can be modified after deployment.

6. Policy Testing & Verification

After linking the GPOs to the appropriate OUs, I tested policy enforcement using:

gpupdate /force gpresult /r

7. Network Configuration

Both machines were configured on the same subnet (255.255.255.0). The Domain Controller was assigned a static IP address.

The client DNS server was set to the Domain Controller IP. Connectivity was verified using nslookup.

8. Joining the Client to the Domain

The client machine was joined to company.local using Domain Controller credentials.

After joining, IT department users can successfully log in using their domain credentials.
