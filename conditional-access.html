Conditional Access & Identity Security (Microsoft Entra ID)

This lab focuses on securing a cloud identity environment using Microsoft Entra ID by implementing Conditional Access policies. The goal was to enforce Zero Trust principles, strengthen authentication, and analyze identity activity through security logs.

Objectives

  • Implement Conditional Access policies based on user risk
  • Enforce multi-factor authentication (MFA)
  • Apply least privilege and RBAC
  • Validate authentication flows
  • Analyze logs for monitoring

Environment & Roles

Sofia (User)

Standard user with limited access

Jordan (IT)

Elevated access with controls

Amelia (Admin)

Privileged account requiring strong security

Conditional Access Policies

All Users: MFA enforced

IT Operations: MFA + session control

Admins: Phishing-resistant MFA

All Users: Block legacy authentication

All Users: Block high-risk sign-ins

Security Monitoring

Sign-in Logs:

  • Authentication success/failure
  • Applied policies per login
  • Session behavior
  • MFA triggered for standard users
  • Multiple policies applied simultaneously
  • Admin login blocked without strong MFA

Audit Logs:

  • Tracked changes
  • Verified admin actions
  • Confirmed policy enforcement

Key Takeaways

  • Conditional Access enables the Zero Trust principle
  • Multiple Conditional Access policies can apply to a single user with stricter controls taking priority
  • Privileged accounts need stronger controls
  • Blocking legacy authentication reduces risk
  • Monitoring is critical for security visibility
