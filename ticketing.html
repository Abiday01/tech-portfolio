IT Support Workflow

This page provides an overview of common IT support workflows, including call handling best practices and a ServiceNow-style incident ticket example. It demonstrates how issues are documented, troubleshooted, and resolved in a professional IT support environment.

Call Handling Guide

Greeting & Readiness

- Greet the user in a friendly, professional way.
- Let them explain what’s going on without interrupting.
- Acknowledge the issue so they know you’re listening.

Gather Information

Ask the right questions to understand the problem and narrow it down.

  • When did the issue start?
  • What were you doing before it happened?
  • When was the last time it worked correctly?

Listen carefully and respond clearly and respectfully.

Reassure the User

- Let the user know you’re looking into the issue.
- Keep them in the loop as you troubleshoot so they don’t feel stuck or ignored.

Troubleshooting

- Use your technical knowledge to figure out what’s causing the issue.
- Start with the simplest fixes first, then move to more complex ones if needed.

  • Example: If Outlook keeps signing out, try signing out and back in before reinstalling the app, since reinstalling takes more time.

Escalation

- If the issue is outside your scope or permissions, escalate it to the appropriate team.
- Make sure to pass along clear notes so the next tech knows what’s already been tried.

Closing the Call

- Confirm whether the issue is resolved.
- Quickly summarize what was done.
- Ask if the user needs help with anything else!

Documentation & Ticket Closure

Document everything clearly, including:

  • What was tried
  • What fixed the issue
  • What to do if it happens again

Close the ticket properly.

Ticketing Systems

Example of incident documentation and resolution using a ServiceNow-style workflow.
The incident examples shown below are mock scenarios created for portfolio purposes to demonstrate ticket documentation, troubleshooting, and resolution workflows.

ServiceNow Incident

Number: INC0010005
State: Closed
Caller: Anna
Channel: Chat
Category: Inquiry / Help
Subcategory: Email
Configuration Item: Microsoft Outlook
Assignment Group: Help Desk
Assigned To: Abida
Priority: 4 – Low
Impact: 3 – Low
Urgency: 2 – Medium

Short Description

Outlook crashes and user unable to send emails after password change.

Description

User reports that after changing their password earlier today via a mobile device, Microsoft Outlook on their work laptop began crashing and signing out automatically. User is unable to send emails.

Additional Comments (Customer Visible)

During troubleshooting:

User (Anna):
Hi, I changed my password on my phone earlier, and now Outlook on my laptop keeps crashing. I also can’t send any emails.

Helpdesk Agent:
Hi Anna, thanks for reaching out. I see that you recently updated your password. Sometimes Outlook requires reauthentication after a password change. I’m going to guide you through signing out and back in.

User:
Okay, sounds good.

Helpdesk Agent:
Please sign out of Outlook completely, then reopen it. When prompted, sign in using your new password. Let me know once you’ve done that.

User:
I signed back in, but it still seems a bit off.

Helpdesk Agent:
Thanks for trying that. I’m going to reconfigure your Outlook profile to sync properly with your updated credentials. This may take a moment.

User:
Okay.

Helpdesk Agent:
All set on my end. Please restart Outlook one more time and try sending a test email.

User:
I just sent a test email and it worked!

Work Notes (Internal)

  • Verified user identity
  • Confirmed password was changed earlier today
  • Observed Outlook crashing and prompting for sign-in
  • Identified cached credentials using old password
  • Guided user through reauthenticating Outlook
  • User confirmed email functionality restored

Resolution Information

Knowledge: ✔️

Resolution Code: Solution Provided

Resolved By: Abida

Resolution Notes:
Issue resolved by signing out of Outlook and signing back in using the updated password. Cached credentials using the old password caused Outlook to crash repeatedly. After reauthentication, Outlook launched successfully and email functionality was restored. User confirmed send and receive is working normally.

Final State: Closed
